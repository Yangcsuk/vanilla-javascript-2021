4.1 Form Submission
  - login form(username)이 비어있어도 안되고, 너무 길어도 안됨(15자 이내)
  - 비어있으면 안되는 내용 :  if (username ===""){ alert("Plaese Write your name"); }
  - 15자 이내 제한방법 : else if (username.length > 15) { alert("Your name is too long"); }
  - 이렇게 user가 입력하는 값의 유효성(validation)을 확인하는 것은 좋다. 왜냐면, user는 항상 이상한 짓을 하기때문
  - 코코아 클론에서 배웠듯이(=너는 무조건 까먹었을거다) input은 form안에 넣어 15자 이내 및 빈칸 방지 기능이 있음
    <input required maxlength="15" type="text" placeholder="What is your name?" />
  - 그런데, form안에 있는 input을 눌러 submit하면 html페이지가 계속 새로고침이 된다. 이걸 다음 강의에서 고쳐보자.

4.2~3 Events
  - form이 submit 되면 html 페이지가 새로고침 되는것은 브라우저의 기본기능이다.
    →!! 우리는 이 기본기능을 멈춰야 한다.!!
  - (중요)function을 선언하고, function이름 뒤에 "(), 작은괄호"를 붙이면 바로 해당 function을 실행해 버린다.
  - addEventListener("특정event", function)
    : 특정event가 발생하면, function에 특정정보(argument)를 담아서 function을 실행시킨다.
  - preventDefault() : 어떤 event의 기본행동이 발생되지 않도록 막는다.(예 : submit 하면 HTML 새로고침 되는것 등..)

4.5 Saving Username
  - 저장장소가 여러가지가 있음(Local Storage, Web SQL, Cookies, Trusted Token...)
  - 이번 강의는 Local Storage API를 사용할 것임.
  - localStorage는 브라우저가 가지고 있는 작은 DB같은 API이다.(API 뜻은 모름ㅋㅋ)
  - localStorage.setItem("key","value") : 로컬스토리지에 키와 값을 저장함.
  - localStorage.getItem("key") -> 키(key)를 호출하면 값(value)을 출력한다.
  - localStorage.removeItem("key") : 로컬스토리지에 key에 해당되는 값을 삭제한다.

6.0 Quotes
  - JS에서 기본으로 제공하는 Math 기능(Function)을 이용하면 다양한 것을 할 수 있다.
  - Math.random() => 0 "이상" 1 "미만" 사이의 무작위(random)값을 생산해줌
  - Math.round() => 소수 첫째 자리에서 반올림(4사5입)해줌
  - Math.ceil() => 소수 첫째 자리에서 올림
  - Math.floor() => 소수 첫재 자리에서 내림
  - Array.length => Array의 길이(n) 값을 반환(return)함, -> quotes.length = 10 (quotes안의 명언 갯수는 10개이다.)
  - Math.floor(Math.random() * quotes.length) = 내림값(floor)을 이용해서 n개의 값을 random하게 선택할 수 있다.
